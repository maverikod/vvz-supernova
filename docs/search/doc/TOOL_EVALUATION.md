# Оценка инструмента поиска теории

Документ составлен на основе анализа функциональности и практического использования (2025-01-01).

---

## 1. Текущий функционал (что уже есть)

### 1.1. Базовый поиск
✅ Поиск по фразам (одна или несколько через `--phrases`)
✅ Поиск по категориям (`--category`)
✅ Поиск по тегам/ID (`--tag`)
✅ Поиск по формулам (`--scope formulas`)
✅ Поиск по сегментам (`--scope segments`)

### 1.2. Управление базой данных
✅ Построение SQLite базы (`sqlite_build`)
✅ Построение цепочки баз (`sqlite_build_chain`) с автоматическим шардированием
✅ Валидация покрытия (`sqlite_validate`)
✅ Экспорт сегментов (`sqlite_export_segment`)

### 1.3. Анализ и статистика
✅ Статистика по индексу (`stats`)
✅ Валидация структуры (`validate`)
✅ Дерево категорий (`tree`)
✅ Сборка сегментов в файл (`assemble`)

### 1.4. Удобство использования
✅ Дедупликация результатов (`--dedupe-by-id`)
✅ Краткий вывод (`--summary-only`)
✅ Пресеты (`--preset earth|sun|particles`)
✅ Форматы вывода (`--format text|json`)
✅ Fallback с FTS на LIKE для кириллицы

## 2. Отсутствующий функционал (что не хватает)

### 2.1. Логические операторы поиска

**Проблема**: Текущий поиск поддерживает только OR (через `--phrases`), но нет AND и NOT.

**Что нужно**:
- Поиск с AND: все фразы должны присутствовать
- Поиск с NOT: исключение фраз
- Комбинированные запросы: `(A OR B) AND (C OR D) NOT E`

**Пример использования**:
```bash
--query "(закон масштаб OR масштабирование) AND (уровень OR гипердефект) NOT ансамбль"
```

### 2.2. Поиск по близости (proximity search)

**Проблема**: Нет возможности искать слова, которые находятся рядом друг с другом.

**Что нужно**:
- Поиск фраз в пределах N слов
- Поиск с учётом порядка слов
- Поиск в пределах одного абзаца/раздела

**Пример использования**:
```bash
--phrase "закон масштаб" --proximity 10
# Найти "закон" и "масштаб" в пределах 10 слов друг от друга
```

### 2.3. Поиск по числовым значениям в формулах

**Проблема**: Нет возможности искать формулы по числовым значениям параметров.

**Что нужно**:
- Поиск формул с конкретными числовыми значениями
- Поиск формул с диапазонами значений
- Поиск формул с приблизительными значениями

**Пример использования**:
```bash
--formula-value "K_*" --range "35:50"
# Найти все формулы, где K_* находится в диапазоне 35-50
```

### 2.4. Поиск по регулярным выражениям

**Проблема**: Нет поддержки регулярных выражений для сложных паттернов.

**Что нужно**:
- Поиск по regex в тексте
- Поиск по regex в формулах
- Поиск по regex в ID/категориях

**Пример использования**:
```bash
--regex "7d-10[0-9].*масштаб"
# Найти все блоки 7d-100-7d-109, содержащие "масштаб"
```

### 2.5. Поиск по контексту (окружающий текст)

**Проблема**: Нет возможности искать с учётом контекста (что находится до/после найденного фрагмента).

**Что нужно**:
- Поиск с контекстом N строк до и после
- Поиск в пределах одного раздела/главы
- Поиск с учётом структуры документа

**Пример использования**:
```bash
--phrase "закон масштаб" --context 50
# Показать 50 строк до и после каждого найденного фрагмента
```

### 2.6. Поиск по ссылкам между блоками

**Проблема**: Нет возможности находить блоки, которые ссылаются друг на друга или связаны тематически.

**Что нужно**:
- Поиск блоков, ссылающихся на конкретный блок (например, "7d-105")
- Поиск связанных блоков по тематике
- Поиск блоков, упоминающих те же ключевые слова

**Пример использования**:
```bash
--references "7d-105"
# Найти все блоки, которые ссылаются на 7d-105
```

### 2.7. Ранжирование по релевантности

**Проблема**: Результаты не ранжируются по релевантности, выдаются в произвольном порядке.

**Что нужно**:
- Сортировка по релевантности (TF-IDF, BM25)
- Учёт частоты вхождения фразы
- Учёт позиции вхождения (заголовок важнее текста)

**Пример использования**:
```bash
--phrase "закон масштаб" --sort relevance
# Отсортировать результаты по релевантности
```

### 2.8. Поиск с подсветкой найденных фрагментов

**Проблема**: В выводе не выделяются найденные фрагменты текста.

**Что нужно**:
- Подсветка найденных фраз в выводе
- Выделение контекста вокруг найденных фрагментов
- Цветовая подсветка для терминала

**Пример использования**:
```bash
--phrase "закон масштаб" --highlight
# Подсветить найденные фрагменты в выводе
```

### 2.9. Поиск по структуре документа

**Проблема**: Нет возможности искать по структуре (заголовки, списки, таблицы).

**Что нужно**:
- Поиск только в заголовках
- Поиск только в списках
- Поиск только в таблицах
- Поиск по уровням заголовков (h1, h2, h3)

**Пример использования**:
```bash
--phrase "закон масштаб" --scope headings
# Искать только в заголовках
```

### 2.10. Поиск с учётом синонимов

**Проблема**: Нет поддержки синонимов и вариантов написания.

**Что нужно**:
- Словарь синонимов для ключевых терминов
- Учёт вариантов написания (кириллица/латиница)
- Учёт сокращений и аббревиатур

**Пример использования**:
```bash
--phrase "закон масштаб" --synonyms
# Искать также "scaling law", "закон подобия", "масштабирование"
```

### 2.11. Визуализация связей между блоками

**Проблема**: Нет возможности визуализировать связи между блоками теории.

**Что нужно**:
- Граф связей между блоками
- Визуализация цитирований
- Визуализация тематических кластеров

**Пример использования**:
```bash
--mode graph --segment-id "7d-105" --output graph.png
# Построить граф связей для блока 7d-105
```

### 2.12. Экспорт в другие форматы

**Проблема**: Поддерживается только text и json, нет экспорта в другие форматы.

**Что нужно**:
- Экспорт в HTML с подсветкой синтаксиса
- Экспорт в PDF
- Экспорт в Markdown с улучшенным форматированием
- Экспорт в LaTeX

**Пример использования**:
```bash
--mode sqlite_export_segment --segment-id "7d-105" --format html --output 7d-105.html
```

### 2.13. Поиск по истории изменений

**Проблема**: Нет возможности отслеживать изменения в блоках теории.

**Что нужно**:
- История изменений блоков
- Поиск по версиям
- Дифф между версиями

**Пример использования**:
```bash
--mode history --segment-id "7d-105"
# Показать историю изменений блока 7d-105
```

### 2.14. Поиск по метаданным

**Проблема**: Нет поиска по датам, авторам и другим метаданным (если они есть).

**Что нужно**:
- Поиск по дате создания/изменения
- Поиск по автору
- Поиск по версии
- Поиск по статусу (черновик, финальная версия)

**Пример использования**:
```bash
--author "Vasiliy Zdanovskiy" --date "2024-01-01:2025-01-01"
```

### 2.15. Интерактивный режим поиска

**Проблема**: Нет интерактивного режима для последовательного уточнения поиска.

**Что нужно**:
- Интерактивная оболочка (REPL)
- История поисковых запросов
- Сохранение и загрузка поисковых запросов
- Автодополнение поисковых фраз

**Пример использования**:
```bash
--mode interactive
# Запустить интерактивный режим поиска
```

### 2.16. Поиск с фильтрами по длине текста

**Проблема**: Нет возможности фильтровать результаты по длине текста.

**Что нужно**:
- Фильтр по минимальной/максимальной длине сегмента
- Фильтр по количеству строк
- Фильтр по количеству слов

**Пример использования**:
```bash
--phrase "закон масштаб" --min-length 1000 --max-length 5000
# Найти только сегменты длиной от 1000 до 5000 символов
```

### 2.17. Поиск с группировкой результатов

**Проблема**: Результаты не группируются по категориям или другим критериям.

**Что нужно**:
- Группировка по категориям
- Группировка по тегам
- Группировка по датам
- Группировка по релевантности

**Пример использования**:
```bash
--phrase "закон масштаб" --group-by category
# Сгруппировать результаты по категориям
```

### 2.18. Поиск с ограничением количества результатов

**Проблема**: Нет явного способа ограничить количество результатов.

**Что нужно**:
- Параметр `--limit N` для ограничения количества результатов
- Параметр `--offset N` для пагинации
- Параметр `--page N --page-size M` для удобной пагинации

**Пример использования**:
```bash
--phrase "закон масштаб" --limit 10 --offset 20
# Показать результаты 21-30
```

### 2.19. Поиск с сохранением результатов

**Проблема**: Нет возможности сохранить результаты поиска для последующего анализа.

**Что нужно**:
- Сохранение результатов в файл
- Сохранение поисковых запросов
- Загрузка сохранённых результатов

**Пример использования**:
```bash
--phrase "закон масштаб" --save-results results.json
# Сохранить результаты в файл
```

### 2.20. Поиск с уведомлениями

**Проблема**: Нет возможности получать уведомления о новых результатах.

**Что нужно**:
- Мониторинг изменений в базе
- Уведомления о новых блоках, соответствующих запросу
- Email-уведомления (опционально)

**Пример использования**:
```bash
--phrase "закон масштаб" --watch --notify email
# Мониторить изменения и отправлять уведомления на email
```

## 3. Приоритеты внедрения

### Высокий приоритет (критично важно)

1. **Логические операторы поиска** (AND, OR, NOT) — необходимо для точного поиска
2. **Ранжирование по релевантности** — улучшает качество результатов
3. **Поиск по близости** — важно для поиска связанных терминов
4. **Ограничение количества результатов** — необходимо для больших результатов
5. **Поиск с подсветкой** — улучшает читаемость результатов

### Средний приоритет (желательно)

6. **Поиск по регулярным выражениям** — расширяет возможности поиска
7. **Поиск по контексту** — полезно для понимания результатов
8. **Поиск по ссылкам между блоками** — важно для навигации
9. **Экспорт в другие форматы** — улучшает интеграцию
10. **Поиск по структуре документа** — полезно для структурированного поиска

### Низкий приоритет (можно отложить)

11. **Визуализация связей** — nice to have
12. **Интерактивный режим** — удобно, но не критично
13. **Поиск по истории изменений** — требует инфраструктуры версионирования
14. **Поиск с уведомлениями** — требует дополнительной инфраструктуры
15. **Поиск с учётом синонимов** — требует словаря синонимов

## 5. Заключение

Инструмент поиска теории имеет **хорошую базовую функциональность**, но **не хватает продвинутых возможностей** для эффективной работы с большим объёмом теоретического материала.

**Критически важные недостатки**:
- Отсутствие логических операторов (AND, NOT)
- Отсутствие ранжирования по релевантности
- Отсутствие ограничения количества результатов
- Отсутствие подсветки найденных фрагментов

**Рекомендуется начать с внедрения функций высокого приоритета**, которые значительно улучшат удобство использования инструмента.
