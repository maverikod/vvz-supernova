ТЕХНИЧЕСКОЕ ЗАДАНИЕ
Тема: сбор и подготовка данных по сверхновым и атомным переходам

ЦЕЛЬ
Собрать два независимых набора данных:
1. каталог атомных переходов
2. каталог вспышек сверхновых

Никаких физических интерпретаций, гипотез и теорий не использовать.
Нужна только техническая подготовка данных для последующего анализа.

ОБЩИЕ ПРАВИЛА
1. Не придумывать данные.
2. Не использовать синтетические значения.
3. Все исходные данные сохранять отдельно от очищенных.
4. Для каждого файла сохранить источник, дату скачивания и ссылку на источник.
5. При отсутствии поля сохранять NaN, а не удалять строку целиком.
6. Все единицы измерения привести к стандартным и явно указать.
7. Все шаги должны быть воспроизводимыми из Python-скриптов.

ЧАСТЬ A. АТОМНЫЕ ПЕРЕХОДЫ

ИСТОЧНИК
Основной источник:
NIST Atomic Spectra Database

ЗАДАЧА
Скачать таблицы атомных спектральных линий для элементов:
H, He, C, N, O, Ne, Na, Mg, Al, Si, P, S, Cl, Ar, K, Ca, Fe, Ni

Если возможно, скачать также данные для всех доступных нейтральных и слабо ионизованных состояний.

НУЖНЫЕ ПОЛЯ
Для каждой спектральной линии сохранить:

element
ion_state
wavelength_vac_nm
wavelength_air_nm
frequency_hz
wavenumber_cm1
Aki_s^-1
intensity
Ei_cm1
Ek_cm1
lower_configuration
upper_configuration
lower_term
upper_term
lower_J
upper_J
line_type
source_catalog
source_url

ОБРАБОТКА
1. Перевести длины волн в частоты:
   frequency_hz = c / wavelength_vac
2. Если есть только air wavelength, явно отметить это.
3. Очистить таблицы от Excel-подобных префиксов, кавычек и нечисловых артефактов.
4. Удалить точные дубликаты строк.
5. Сохранить исходные данные отдельно.
6. Сохранить объединённую очищенную таблицу.

ИТОГОВЫЕ ФАЙЛЫ

data/atomic_lines_clean.csv

ЧАСТЬ B. СВЕРХНОВЫЕ

ИСТОЧНИКИ
Скачать данные из открытых каталогов сверхновых. Приоритет:
1. Open Supernova Catalog
2. ASAS-SN
3. ZTF public supernova / transient tables
4. Pan-STARRS public SN tables
5. Другие открытые каталоги, если в них есть light-curve и дата максимума

НУЖНЫЕ ПОЛЯ
Для каждой сверхновой сохранить:

peak_mjd
peak_mag
band
distance_modulus
luminosity_distance_Mpc

Если доступны кривые блеска, для каждой точки:

sn_name
mjd
mag
mag_err
flux
flux_err
band
instrument
source_catalog

ОБРАБОТКА
1. Скачать и сохранить light-curves в длинном формате.
2. Для каждой сверхновой попытаться вычислить:
   rise_time_days
   decay_time_days
   peak_width_days
   peak_flux
3. Если расчёт невозможен, сохранить NaN.
4. Сохранить тип сверхновой как есть из каталога, не переименовывать вручную.
5. Удалить точные дубликаты.
6. Оставить raw-версии отдельно.

ЧАСТЬ C. ЕДИНИЦЫ И ФОРМАТЫ

СТАНДАРТ ЕДИНИЦ
Атомные данные:
- wavelength: nm
- frequency: Hz
- wavenumber: cm^-1
- Aki: s^-1

Сверхновые:
- time: MJD
- durations: days
- luminosity distance: Mpc
- flux: как в исходном каталоге, но единица должна быть сохранена в отдельном поле, если доступна

ЧАСТЬ D. КОНТРОЛЬ КАЧЕСТВА

НУЖНО ВЫВЕСТИ
1. Число элементов, по которым реально собраны атомные линии.
2. Общее число атомных линий.
3. Число сверхновых в каталоге.
4. Число сверхновых с light-curve.
5. Число сверхновых, для которых удалось оценить rise_time_days.
6. Список источников, реально использованных в итоговом наборе.
